50 万以内资金规模适配的 A 股日频多因子选股系统

本项目不做程序化下单、不做日内高频、不做 CTA。V1 策略定位为长多日频多因子选股：用本地增量数据计算全 A 股票综合得分，选出 Top5 目标持仓，用 Top15 缓冲降低换手，并输出 9:30 后可人工执行的交易计划。

目标与非目标

系统最终服务的是人工交易决策，而不是自动下单。所有输出必须可解释、可回测、可复现，并保留来源口径。当前资金规模按 50 万以内设计，持仓上限 5-10 只，优先降低换手和人工执行复杂度。

每天必须输出

  • Top5 目标持仓、Top10/Top15 缓冲榜和全量股票综合得分。
  • 保留、买入、卖出、跳过四类操作建议。
  • 建议成交价格区间，不给单点神奇价格。
  • 取消条件：涨跌停、停牌、流动性不足、权限不可交易、盘口过薄。

系统必须解释

  • 每个标的的综合分、因子贡献和排名变化。
  • 参与打分的因子、权重、标准化方式和数据源。
  • 因子近期 IC、ICIR、分组收益、相关性和换手成本。
  • 本次调仓原因：进入 Top5、跌出 Top15、仍在缓冲区或不可交易。

明确不做

  • 不接券商自动下单接口。
  • 不做 CTA、日内高频、机器学习排序和动态 Softmax 权重。
  • 不在 V1 做数十至数百只股票的机构式指数增强。
  • 不每天重复拉取全市场历史 K 线。
  • 不承诺收益，只验证信号质量和执行可行性。

V1 策略框架

本项目当前只落地一套策略：A 股长多日频多因子选股。多因子负责“选什么”，组合规则负责“持哪些”，执行模块负责“什么价格区间手动买卖”。

1每日增量更新收盘后或次日 9:00 前补齐日线、复权、分钟聚合、盘口快照。
2股票池过滤剔除 ST、停牌、新股、权限不可交易、低流动性和涨跌停不可交易标的。
3因子打分计算 active 因子，去极值、中心化、标准化并统一方向。
4综合排名V1 使用等权合成，不使用 Softmax 动态权重。
5组合构建Top5 是目标持仓，Top15 是卖出缓冲，最多持仓 5 只。
6人工执行9:30 后按交易计划买入、卖出、保留或跳过。

买卖规则

  • 初始无持仓时，买入综合分 Top5 中满足可交易条件的股票。
  • 已有持仓进入新一日 Top15，继续保留；跌出 Top15 才触发卖出候选。
  • 新股票进入 Top5 且组合有空位或卖出候选释放仓位，才进入买入清单。
  • 当日涨停、停牌、流动性不足或权限不可交易时跳过，并保留跳过原因。

仓位与约束

  • 资金规模按 50 万以内设计，默认最多 5 只，每只目标仓位约 20%。
  • 手动交易下 Top10 可作为候选扩展，Top15 作为缓冲边界。
  • A 股 T+1 约束必须进入回测和交易计划，买入当日不可卖出。
  • 不满 5 只时允许留现金，不强制买入低质量或不可执行标的。

因子基础

因子是同一天跨股票可比较的一列特征。多因子模型把多个弱信号标准化、合成、排序，再通过组合规则和回测把排序转化成可执行持仓。V1 不追求复杂模型，先建立因子准入、因子池、组合增量验证和可视化复盘机制。

量价因子

来自开高低收、成交量、成交额、VWAP、盘口、分钟线。Alpha191 主要属于这一类，适合短周期信号。

AI 辅助因子挖掘

  • AI 只能提出候选因子定义、经济含义、依赖字段和计算公式。
  • 候选因子必须先进入 candidate，完成覆盖率、IC、ICIR、分组收益、相关性和成本评估后才能升级。
  • 1m/5m/盘口数据先聚合为日频微观结构因子，例如尾盘强度、VWAP 偏离、成交量结构和盘口压力。
  • 如果只有盘口快照，不得描述为逐笔订单流，只能描述为盘口快照派生因子。

因子池状态

  • candidate：刚实现，只做研究。
  • watch：表现尚可，但稳定性、相关性或成本仍需观察。
  • active：进入综合打分，V1 默认 active 因子等权。
  • paused：近期失效，暂不参与打分。
  • retired：长期失效或被更优因子替代。

组合增量原则

单因子 IC、ICIR 表现优秀，不代表加入组合后一定提升收益。新增因子必须证明它提供了不同于现有 active 因子的新信息，并在 Top5/Top15 组合约束、T+1、交易成本和手动执行约束下改善整体表现。

数据底座设计

接口联调与能力核验已完成。全市场 5000+ 股票不能每天重复拉取全历史 K 线；系统必须区分首次 bootstrap、每日增量更新、缓存命中、分页拉取和数据质量校验。

1confirmed adapters接口已支持单标的指定时段、多标的单日批量、offset + limit 分页。
2bootstrap首次拉取历史日线、复权、基础信息和必要分钟窗口，写入本地 raw cache。
3daily incremental每日只补缺失交易日和变更字段，优先走多标的单日批量。
4normalize字段、单位、主键、市场代码、复权口径统一。
5factor mart量价、基本面、盘口快照派生日频因子宽表。
6signal martUI、回测、每日信号读取，所有输出保留 source 与 freshness。

缓存与增量规则

  • 原始接口返回按 provider/api_name/date 分区保存，便于复现和排查。
  • 标准化数据落 DuckDB/Parquet，本地查询不反复打外部接口。
  • 每日任务先查本地最大 trade_date，只拉缺失交易日。
  • 当日或昨日行情优先使用多标的单日批量查询；长周期历史数据使用单标的分页查询。

已确认接口能力

  • 单标的查询：支持按代码获取指定时段数据。
  • 分钟 K 线：支持拉取单个标的昨日全天 9:30-15:00 的 1m K 线。
  • 日 K 线：支持一次性获取单只标的 500 天、1000 天日 K 线。
  • 海量数据：支持 offset + limit 分页分批拉取。
  • 多标的单日查询：支持传入数千个标的代码，统一获取同一交易日行情数据。

日内数据在 V1 的使用边界

TDX 1m/5m 与五档盘口先用于流动性判断、成交区间生成和日频微观结构因子探索，不做日内高频交易。所有日内特征必须在收盘后聚合成按 trade_date 对齐的日频因子，再进入次日 9:00 的综合打分。

V1 因子池初始化

第一批 active 因子只使用低算子复杂度的 Alpha191 量价因子，避免在数据和评估框架未稳定前铺开 191 个公式。AI 挖掘出的日内微观结构因子先进入 candidate，不直接参与打分。

因子公式需要算子方向处理
Alpha014CLOSE - DELAY(CLOSE, 5)DELAY由 IC 自动判定正反向。
Alpha015OPEN / DELAY(CLOSE, 1) - 1DELAY开盘跳空，先按原公式。
Alpha018CLOSE / DELAY(CLOSE, 5)DELAY5 日收益比例。
Alpha020(CLOSE - DELAY(CLOSE, 6)) / DELAY(CLOSE, 6) * 100DELAY6 日动量。
Alpha024SMA(CLOSE - DELAY(CLOSE, 5), 5, 1)DELAY, SMASMA 采用中国式递推定义。
Alpha029(CLOSE - DELAY(CLOSE, 6)) / DELAY(CLOSE, 6) * VOLUMEDELAY量价结合。
Alpha031(CLOSE - MEAN(CLOSE, 12)) / MEAN(CLOSE, 12) * 100MEAN价格偏离均线。
Alpha034MEAN(CLOSE, 12) / CLOSEMEAN均线/现价。
Alpha046(MEAN(CLOSE,3)+MEAN(CLOSE,6)+MEAN(CLOSE,12)+MEAN(CLOSE,24))/(4*CLOSE)MEAN多周期均线位置。
Alpha053COUNT(CLOSE > DELAY(CLOSE,1), 12) / 12 * 100COUNT, DELAY上涨天数占比。

算子验收

  • DELAY、MEAN、SMA、COUNT 有单元测试。
  • SMA(X,N,M) 使用 Y=(M*X+(N-M)*Y_prev)/N。
  • 所有因子支持按交易日截面输出，缺失值不静默填 0。

研究验收

  • 每个因子输出覆盖率、Rank IC、ICIR、五分组收益、换手率。
  • 支持未来 1/3/5/10 日收益评估。
  • 因子方向由训练窗口 IC 符号确定，样本外不得重看未来。

验收标准

每个阶段都必须能被命令、文件和 UI 共同验证。不能只靠“看起来能跑”。

数据验收

  • 所有表有主键唯一性检查、覆盖率、缺失率、最近刷新时间。
  • 全市场历史数据只在 bootstrap 阶段拉取，日常任务只做增量更新。
  • 当日/昨日行情使用多标的单日批量查询；长周期历史 K 线使用单标的分页查询。
  • offset + limit 分页必须能断点续跑，并记录每批拉取状态。
  • TDX 1m/5m 聚合日线能与 Tushare daily 对账。
  • 东财接口统一节流，失败输出 source unavailable，不伪装为空结果。
  • Tushare token 只从环境或配置读取，日志中永不显示。

因子验收

  • 10 个 Alpha191 因子都有单元测试和快照样例。
  • 每个因子都有 Rank IC、ICIR、覆盖率、分组收益、相关性、换手成本。
  • 因子池支持 candidate、watch、active、paused、retired 状态。

  • 训练期确定方向，样本外验证，不得未来函数。
